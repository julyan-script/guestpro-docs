# GuestPro Open API Documentation

Welcome to GuestPro Open API Documentation

This page contains information about the services and API provided by GuestPro. You'll find API documentation with examples, object model explanations, code snippets and integration best practices examples.

## API Reference

The GuestPro API is organized around REST. Our API has predictable, resource-oriented URLs, and uses HTTP response codes to indicate API errors. We use built-in HTTP features and HTTP verbs, which are understood by off-the-shelf HTTP clients. JSON is returned by all API
responses, including errors.

API support `GET`, `POST`, `PUT` and `DELETE` queries.

Each response is valid JSON object and **MUST** contain at least one key: `success`, `message` or `data`.

If response has success status, it **MUST** contain success and true value at response object.

```jsx title="Plain text"
{
  "Success": true,
  "message": "Reservation success"
}
```
If response has fai status, it **MUST** contain `success` and `false` value at response object.

```jsx title="Plain text"
{
  {
  "Success": false,
  "message": "Reservation id not found"
  }
}
```
All the API requests should be made over HTTPS instead of HTTP (all calls made over plain HTTP will fail)

## Authentication
To let you explore our APIs, make sure that you have registered an account. We provide you `username`, `password`, and `company_code` for both the development and production environments.

You have to include your credentials (`username`, `password`, and `company_code`) in every api request. Your credentials should be kept private so do not share your credentials.

:::note[Note]

You can get the company code after subscription.

:::

## Headers
Each request and response only in JSON format, use this content-type

```bash
Content-Type application/json 
```

## Error & Status Code
The GuestPro API uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicates success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted, validation errors, etc.). Codes in the 5xx range indicates an error with the GuestPro servers and you should retry.

`200 OK` Success response.

`404 Not Found` The requested resource doesn't exist.

`500 Internal Server Error` The server has encountered a situation it does not know how to handle.
